📡 Stream Health Check

Recommends a bitrate/resolution from your actual measured upload speed, and tests whether your hardware video encoder is really performing better than plain CPU encoding — instead of just guessing from your computer's specs. Runs automatically once OBS is detected; briefly starts and stops a short throwaway test recording in OBS to measure it (never while the real stream is live).

Live quality scaling

OBS's dynamic bitrate (enabled automatically by setup) flexes with the connection seamlessly. If the line struggles badly for a sustained period, quality can also be stepped down a ladder — each step briefly restarts the stream (viewers see a few seconds of buffering; the broadcast survives). Requires streaming with a plain stream key — OBS's connected-YouTube-account mode with auto-stop ENDS the broadcast when the stream stops, so a quality step would kill it. In YouTube Studio copy the stream key and paste it into OBS Settings → Stream.

YouTube Broadcast

Sets your live broadcast's title, description, privacy, category and "made for kids" flag over the YouTube API — the settings you'd normally use OBS's Manage Broadcast panel for, which disappears when you stream with a stream key. Requires the one-time Google OAuth setup below.

{home} and {away} are replaced with the team names automatically

{home} and {away} work here too

"Made for kids" can't be changed here — YouTube only allows it when the broadcast is created. Set it in YouTube Studio → Go Live (grassroots cricket is normally "No, not made for kids").

One-time setup:
1. Go to console.cloud.google.com
2. Create a project → Enable YouTube Data API v3
3. Create OAuth2 credentials (Desktop app) → Download JSON
4. Rename file to yt_credentials.json and place in the same folder as server.py
5. Run: pip install google-api-python-client google-auth-oauthlib
6. Click the button above — a browser window will open to authorise once
Tip: create the broadcast first (YouTube Studio → Go Live → Stream), then click Update to fill in its details.

📊 Match data

Every ball is logged to a local database (match_data.db) as you stream — your own season-long dataset. The current over is rewritten live so scorer edits are captured; "Reconcile" then pulls the final scorecard from PlayCricket as the authoritative record.
